## Step 4 ‚Äî Create `apps/api/src/app/prisma/prisma.service.ts`

**File:** `apps/api/src/app/prisma/prisma.service.ts` (new file ‚Äî create the directory first)

The `PrismaService` extends the Phase 2 generated `PrismaClient` (imported from
`@prisma/client`), implements `OnModuleInit` to connect, and implements `OnModuleDestroy`
to disconnect gracefully on shutdown.

```typescript
import { Injectable, OnModuleDestroy, OnModuleInit } from '@nestjs/common';
import { PrismaClient } from '@prisma/client';

@Injectable()
export class PrismaService
  extends PrismaClient
  implements OnModuleInit, OnModuleDestroy
{
  async onModuleInit(): Promise<void> {
    await this.$connect();
  }

  async onModuleDestroy(): Promise<void> {
    await this.$disconnect();
  }
}
```

**Checklist before saving:**

- `PrismaClient` is imported from `@prisma/client` (classic generator path ‚Äî **not** from a
  `generated/` directory). This is the import path confirmed by Phase 2 Handoff deviation #1.
- `@Injectable()` decorator is present.
- `extends PrismaClient` is in the class declaration.
- `onModuleInit` calls `this.$connect()`.
- `onModuleDestroy` calls `this.$disconnect()` (graceful shutdown ‚Äî satisfies acceptance
  criterion 2's requirement that the connection is closed on shutdown).
- The `userPreference` and `savedLocation` typed accessors are inherited from `PrismaClient`
  ‚Äî no manual declaration needed.
- No test file is added (testing is Phase 6).

**Verify:**

```powershell
npm run build
```

Expected: exits 0. The webpack build must compile the new file without TypeScript errors.
If `@prisma/client` is not found, confirm `npx prisma generate` was run in Phase 2 and
the generated client is in `node_modules/.prisma/client`.
